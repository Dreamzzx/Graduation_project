### 2.3 模块详解

#### 2.3.1 VideoSource (视频采集模块)

**文件**: `src/video_source/VideoSource.h`, `VideoSource.cpp`

**功能**:
- 使用 DirectShow (dshow) 从摄像头采集视频
- 支持 MJPG 格式采集
- CUDA 硬件解码 (mjpeg_cuvid)

**关键类**:
```cpp
class VideoSource {
public:
    bool Init(const CaptureConfig& config, VideoFrameQueue<cv::Mat>* queue);
    void start();        // 启动采集线程
    void stop();         // 停止采集
    
private:
    AVFormatContext* format_ctx = nullptr;   // 输入格式上下文
    AVCodecContext* codec_ctx = nullptr;      // 解码器上下文
    AVBufferRef* hw_device_ctx = nullptr;     // CUDA 设备上下文
    AVPixelFormat hw_pix_fmt;                // 硬件像素格式
    
    VideoFrameQueue<cv::Mat>* frame_queue;   // 帧输出队列
    std::atomic<bool>* is_running;           // 运行状态标志
    int capture_framerate;                    // 采集帧率
};
```

**采集流程**:
```
1. avformat_open_input() 打开 DirectShow 设备
2. av_find_best_stream() 查找视频流
3. avcodec_find_decoder_by_name("mjpeg_cuvid") 查找 CUDA 解码器
4. av_hwdevice_ctx_create() 创建 CUDA 设备上下文
5. 循环: av_read_frame() -> avcodec_send_packet() -> avcodec_receive_frame()
6. HWFrameToCvMat() 将 GPU 帧转为 OpenCV Mat
7. frame_queue->Push() 推入队列
```

#### 2.3.2 YOLOv11Detector (目标检测模块)

**文件**: `src/core_onnx/yolov11_onnx.h`, `yolov11_onnx.cpp`

**功能**:
- 基于 ONNX Runtime GPU 加速推理
- YOLOv11n 目标检测模型
- 实时检测人物

**关键类**:
```cpp
class YOLOv11Detector {
public:
    bool Init(const DetectorConfig& config,
              VideoFrameQueue<cv::Mat>* input_queue,
              VideoFrameQueue<FrameData>* output_queue);
    void start();    // 启动检测线程
    void stop();     // 停止检测
    
private:
    std::unique_ptr<Ort::Env> ort_env;      // ONNX Runtime 环境
    std::unique_ptr<Ort::Session> session;  // 推理会话
    std::vector<std::string> input_names;   // 输入节点名
    std::vector<std::string> output_names;  // 输出节点名
    
    VideoFrameQueue<cv::Mat>* input_queue;   // 输入队列
    VideoFrameQueue<FrameData>* output_queue; // 输出队列
    
    int input_size = 640;     // 模型输入尺寸
    float scale = 1.0f;      // 缩放比例
    int pad_w = 0, pad_h = 0; // 边框填充
};
```

**推理流程**:
```
1. preprocess() 预处理:
   - cv::resize() 缩放到 640x640
   - cv::copyMakeBorder() 填充到正方形
   - cv::split() 分离通道
   - 归一化到 0-1

2. session->Run() ONNX 推理

3. postprocess() 后处理:
   - 解析输出张量
   - 非极大值抑制 (NMS)
   - 坐标转换回原始图像尺寸

4. draw_detections() 绘制检测框:
   - 直接在原图绘制 (避免 clone)
```

#### 2.3.3 FFmpegPush (RTSP 推流模块)

**文件**: `src/core_push/video_push.h`, `video_push.cpp`

**功能**:
- NVENC GPU 硬件编码
- RTSP 协议推流
- MPEG 90kHz 时间基

**关键类**:
```cpp
class FFmpegPush {
public:
    bool Init(const char* url, VideoFrameQueue<FrameData>& queue,
              int width, int height, int bitrate, int fps);
    void pushFrame(FrameData frame_data);  // 推入一帧
    
private:
    AVFormatContext* fmt_ctx = nullptr;   // 输出格式上下文 (rtsp)
    AVCodecContext* codec_ctx = nullptr;  // 编码器上下文
    const AVCodec* codec = nullptr;       // 编码器 (h264_nvenc)
    
    AVBufferRef* hw_device_ctx = nullptr; // CUDA 设备
    AVBufferRef* hw_frames_ctx = nullptr; // CUDA 帧上下文
    
    SwsContext* sws_ctx = nullptr;        // 色彩空间转换
    AVFrame* sw_frame = nullptr;          // 软件帧 (NV12)
    AVFrame* hw_frame = nullptr;          // GPU 帧 (CUDA)
    
    int64_t current_pts = 0;              // 当前 PTS
    int64_t start_time_ms = 0;          // 开始时间 (毫秒)
    
    // 时间基: MPEG 90kHz
    const AVRational src_time_base = {1, 90000};
};
```

**推流流程**:
```
1. avformat_alloc_output_context2(&fmt_ctx, nullptr, "rtsp", url)
2. avcodec_find_encoder_by_name("h264_nvenc") 查找 NVENC
3. 配置编码器:
   - preset: p1 (最低延迟)
   - tune: ull (超低延迟)
   - rc: cbr (恒定码率)
   - gop_size: fps (I帧间隔)
4. av_hwdevice_ctx_create() 创建 CUDA 设备
5. av_hwframe_ctx_alloc() 分配 CUDA 帧缓冲

6. pushFrame() 每帧处理:
   - convert_bgr_to_nv12() BGR24 → NV12
   - av_hwframe_transfer_data() CPU → GPU
   - avcodec_send_frame() 发送到编码器
   - avcodec_receive_packet() 获取编码包
   - av_write_frame() 写入 RTSP 流
```

**PTS 时间戳设计**:
```cpp
// 使用实际时间戳，避免累积延迟
auto now_ms = std::chrono::duration_cast<std::chrono::milliseconds>(
    std::chrono::steady_clock::now().time_since_epoch()).count();

if (start_time_ms == 0) start_time_ms = now_ms;

int64_t elapsed_ms = now_ms - start_time_ms;
current_pts = elapsed_ms * 90;  // MPEG 90kHz: 1ms = 90 ticks
```

#### 2.3.4 VideoRecorder (本地录制模块)

**文件**: `src/core_recorder/VideoRecorder.h`, `VideoRecorder.cpp`

**功能**:
- 检测到人物自动开始录制
- 人物离开后自动停止
- MP4 封装

**关键类**:
```cpp
class VideoRecorder {
public:
    bool Init(const RecordingConfig& config);
    void onPersonDetected(const cv::Mat& frame);  // 检测到人
    void onPersonLeft();                            // 人离开
    bool isRecording() const;                      // 是否在录制
    
private:
    bool startRecording();    // 开始录制
    bool writeFrame();       // 写入帧
    
    AVFormatContext* fmt_ctx = nullptr;   // MP4 输出
    AVCodecContext* codec_ctx = nullptr; // 编码器
    AVStream* video_stream = nullptr;    // 视频流
    
    bool use_hw_encoder = false;          // 是否使用 GPU 编码
    AVBufferRef* hw_device_ctx = nullptr; // CUDA 设备
    AVBufferRef* hw_frames_ctx = nullptr; // CUDA 帧
};
```

**录制流程**:
```
1. onPersonDetected() 被调用:
   - 如果未录制，startRecording()
   - 写入当前帧

2. onPersonLeft() 被调用:
   - 启动超时计时器
   - 超时后 stopRecording()

3. startRecording():
   - avformat_alloc_output_context2(..., "mp4", filename)
   - avcodec_find_encoder_by_name("h264_nvenc") GPU 编码
   - avio_open() 打开输出文件
   - avformat_write_header() 写文件头

4. writeFrame():
   - sws_scale() BGR24 → NV12
   - av_hwframe_transfer_data() CPU → GPU
   - avcodec_send_frame() 编码
   - av_interleaved_write_frame() 写入

5. stopRecording():
   - 刷新编码器剩余数据
   - av_write_trailer() 写文件尾
   - 释放资源
```

#### 2.3.6 VideoFrameQueue (线程安全队列)

**文件**: `src/safe_queue/VideoFrameQueue.h`

**功能**:
- 线程安全的帧队列
- 支持阻塞/非阻塞获取
- 可配置队列大小

```cpp
template<typename T>
class VideoFrameQueue {
public:
    explicit VideoFrameQueue(size_t max_size = 2);
    
    void Push(T&& data);           // 推入数据
    std::optional<T> Pop(bool& is_running);  // 阻塞获取
    std::optional<T> PopLatestNonBlocking(bool& is_running);  // 非阻塞获取最新帧
    
    bool IsEmpty() const;
    size_t Size() const;
};
```

### 4.2 页面功能

#### 4.2.1 推流管理页面

**功能**:
- 视频采集配置 (设备、分辨率、帧率)
- RTSP 推流配置 (地址、分辨率、帧率、码率)
- 目标检测配置 (启用/禁用、模型、参数)
- 本地录制配置 (启用/禁用、目录、超时)

**API 调用**:
- `GET/POST /api/config` - 获取/保存配置
- `GET /api/devices` - 获取设备列表
- `GET /api/device_capabilities/<device>` - 获取设备参数

#### 4.2.2 推流预览页面

**功能**:
- 视频流预览 (HTTP-FLV / HLS)
- 连接/断开控制

**API 调用**:
- `GET /api/stream_url` - 获取推流地址

**播放器**:
- flv.js - HTTP-FLV 播放
- hls.js - HLS 播放

#### 4.2.3 日志分析页面

**功能**:
- 实时显示推流程序日志
- 自动刷新 (每 3 秒)

**API 调用**:
- `GET /api/log` - 获取日志

#### 4.2.4 系统资源页面

**功能**:
- CPU 使用率
- 内存使用
- 磁盘使用
- GPU 使用 (NVIDIA)
- 推流进程信息 (CPU、内存、线程)

**API 调用**:
- `GET /api/system` - 获取系统资源 (每 2 秒)

#### 4.2.5 录制文件页面

**功能**:
- 录制文件列表
- 本地播放 (调用系统播放器)
- 删除文件

**API 调用**:
- `GET /api/recordings` - 获取录制列表
- `POST /api/recording/play` - 播放文件
- `POST /api/recording/delete` - 删除文件

### 4.3 启动/停止流程

```
启动流程:
1. 用户点击"启动推流"
2. 如果"自动启动服务器"勾选:
   - POST /api/server/start {server_id: "mediamtx"}
   - 等待 2 秒
3. POST /api/start {start_server: true, server_id: "mediamtx"}
4. 启动 VideoPush.exe 子进程
5. 进程读取 config.json 开始推流

停止流程:
1. 用户点击"停止推流"
2. POST /api/stop {stop_server: false}
3. 终止 VideoPush.exe 进程
4. 更新状态显示
```

## 七、延迟分析

### 7.1 延迟组成

| 处理环节 | 无推理 | 有推理 | 说明 |
|----------|--------|--------|------|
| 视频采集 | 10-20ms | 10-20ms | DirectShow + CUDA |
| CUDA 解码 | 5-10ms | 5-10ms | mjpeg_cuvid |
| YOLOv11 推理 | - | 150-300ms | ONNX GPU |
| 图像缩放 | 5-10ms | 5-10ms | CUDA |
| NVENC 编码 | 10-20ms | 10-20ms | h264_nvenc |
| 网络传输 | 50-100ms | 50-100ms | 局域网 |
| 服务器缓冲 | 50-150ms | 50-150ms | MediaMTX |
| **总计** | **150-350ms** | **300-650ms** | |

### 7.2 延迟优化措施

1. **PTS 实时同步**: 使用实际时间戳，避免累积延迟
2. **队列大小=1**: 减少帧缓冲
3. **NVENC ull**: preset=p1, tune=ull
4. **writeQueueSize=16**: 减少服务器缓冲
